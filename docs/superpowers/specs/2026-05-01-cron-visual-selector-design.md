# Cron 可视化选择器设计文档

**日期：** 2026-05-01  
**作者：** Claude + yuchao.cao  
**状态：** 已审批

---

## 背景

`ScheduleView.vue` 的录制计划表单目前要求用户手动输入原始 Cron 表达式（如 `0 2 * * *`），对非技术用户极不友好。本设计将其替换为三模式可视化选择器，同时保留高级用户所需的完整 Cron 灵活性。

---

## 目标

- 普通用户无需了解 Cron 语法即可完成配置
- 进阶用户可自由选择时间和星期
- 高级用户仍可直接输入原始 Cron 表达式
- 所有模式实时显示人类可读描述和生成的 Cron 值

---

## 方案：三 Tab 模式选择器

表单中「Cron 表达式」字段替换为一个新的 `CronSelector` 组件，内含三个模式 Tab：

### Tab 1 — 快速选择

6 张预设卡片，点击即选，无需任何输入：

| 显示名称 | Cron 值 |
|----------|---------|
| 每天凌晨 2:00 | `0 2 * * *` |
| 工作日早 8:00 | `0 8 * * 1-5` |
| 每天晚上 22:00 | `0 22 * * *` |
| 每 30 分钟 | `*/30 * * * *` |
| 周末凌晨 | `0 0 * * 6,0` |
| 每小时整点 | `0 * * * *` |

选中卡片高亮（紫色描边 + 背景）；未选为灰色。

### Tab 2 — 自定义时间

- **时间选择器**：`<el-time-picker>` 选择小时:分钟
- **星期多选**：7 个圆形按钮（一~日），可任意勾选，全选 = 每天
- 根据时间 + 星期实时生成 Cron 表达式
- 例：`02:00` + 周一~周五 → `0 2 * * 1-5`

### Tab 3 — 高级 (Cron)

- 保留原始 `<el-input>` 文本框，直接输入 Cron 表达式
- 显示格式说明：`分 小时 日 月 周几（5 字段）`

### 共享实时预览条

三种模式底部均显示统一的预览条：

```
✓  每天凌晨 2:00 触发  （0 2 * * *）
```

- 绿色图标 + 描述文字 + 灰色括号内显示 Cron 原文
- Cron 无效时变为红色警告

## 组件设计

### `CronSelector.vue`（新建）

**Props：**
- `modelValue: String` — 当前 Cron 表达式（v-model）

**Emits：**
- `update:modelValue` — 任一模式修改时触发

**内部状态：**
- `activeTab: 'preset' | 'custom' | 'advanced'`
- `customTime: String`（HH:mm）
- `customDays: number[]`（0-6，0=周日）
- 切换 Tab 时不丢失其他 Tab 的状态

**Cron 生成逻辑（自定义时间 Tab）：**
```
days 为空 → '* * * * *'
days 长度 == 7 → `{min} {hour} * * *`
否则 → `{min} {hour} * * {days.join(',')}`
```

**人类可读描述函数：**
- 解析常见 Cron 模式 → 中文描述
- 无法识别时显示"自定义：`{cron}`"

### `ScheduleView.vue`（修改）

- 将表单中的 `el-input cron_expr` 替换为 `<CronSelector v-model="form.cron_expr" />`
- 移除原有的 cron-hint 说明文字（由组件内部承担）

## 不在范围内

- 日 / 月 字段的可视化（极少使用，高级 Tab 已覆盖）
- Cron 表达式的完整验证（后端负责）
- 多语言支持
